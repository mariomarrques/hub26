# Suppliers UI Polish

## Goal
Improve the visual quality, clarity, and responsiveness of the suppliers experience without changing the core product scope.

## Scope
This phase focuses only on the suppliers experience:
- suppliers page
- supplier cards
- supplier details dialog
- supplier review dialog
- supplier social links/actions

## UX goals
- Make the suppliers page feel cleaner and more curated
- Improve visual hierarchy and scanability
- Make supplier details easier to understand
- Turn supplier social/contact links into visually meaningful buttons
- Improve responsiveness on smaller screens
- Preserve current functionality and data flow

## Product decisions
- Keep the page title as **Fornecedores**
- Keep the main actions as:
  - Detalhes
  - Avaliar
- Social/contact actions should appear as individual buttons by type
- Supported visible link types:
  - WhatsApp
  - Grupo
  - Instagram
  - Site
- Show a social button only if that link exists for the supplier
- Do not redesign the whole app
- Do not add new business features in this phase

## Visual direction
- Clean
- Modern
- Premium but simple
- Better spacing and hierarchy
- Consistent action styles
- Better mobile behavior

## Card improvements
- Clearer top section with avatar, name, country, rating
- Better metadata layout for shipping method / prep time / shipping time
- Better action separation between Detalhes and Avaliar

## Details dialog improvements
- Stronger header hierarchy
- Better information grouping
- Description block
- Social links block with icons and recognizable styling
- Cleaner review list presentation

## Review dialog improvements
- Better spacing and hierarchy
- Clearer star rating interaction
- Better file upload clarity
- Clear success feedback

## Non-goals
- No admin redesign
- No supplier suggestion flow
- No ranking system
- No advanced search/filter redesign
- No analytics
